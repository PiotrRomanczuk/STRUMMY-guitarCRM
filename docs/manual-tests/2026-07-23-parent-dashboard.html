Parent Dashboard — family portal (net-new)

Date: 2026-07-23 · Role: parent (is_parent) · integrated to main

1. What was built

A net-new parent-role dashboard implementing the batch-02 "Family portal" mockup: a parent checks in on their child.

  • Child check-in (name, instrument, level, teacher), recent practice (on-track this week, minutes, goal/day), upcoming lessons, and the latest teacher note.
  • Reuses child data via a dedicated parent-health.{helpers,queries} module (streak, week summary, upcoming lessons). Parent-read RLS migration 20260723150000_parent_read_repertoire_preferences.sql.
  • Billing is stubbed (parked per current scope) — no Stripe/invoice logic.

2. How to manually test

  1. npm run dev. Sign in as a user with the parent role that is linked to at least one child.
  2. Confirm the dashboard renders the child check-in, practice summary (on-track vs needs-attention), upcoming lessons, and the latest teacher note.
  3. Confirm the billing section shows only a disabled placeholder (no live billing).
  4. Isolation check (important): confirm a parent sees only their linked child's data — never another student's.
