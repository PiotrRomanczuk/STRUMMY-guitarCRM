Real bugs found and fixed while testing

Sync-conflict diff always showed — for the local side. getPendingConflicts() only ever selected lesson.teacher_id from its join — never title/scheduled_at/notes — so the "Local (Strummy)" column of the conflict-resolution UI was permanently blank, for every conflict, always. Fixed the select; the diff now renders real local-vs-remote data.
File: lib/services/sync-conflict-resolver.ts
Practice streak / week-bounds broke for ~2 hours a day in non-UTC timezones. computePracticeStreakDays mixed UTC-normalized practice dates (from the DB) with a locally-computed "today" anchor; getMonToSunWeekBounds read now.getDay() (local) but did all its arithmetic in UTC. Both broke whenever local and UTC disagreed on the calendar day (e.g. it's after midnight locally but not yet midnight UTC) — reproduced live during this session. Made both functions consistently UTC-based.
File: app/actions/student/dashboard.helpers.ts

Local dev-environment gaps found and fixed (StudentManager stack on uwh)

The dev DB was missing most of the schema — 7 tables existed against the repo's ~62. Ran the full supabase/migrations/ set against it: 7 → 67 tables. Fixed missing grants on newly-created tables and added default privileges so this doesn't recur.

52 profiles (including the teacher/admin/student E2E fixture accounts) had profiles.id ≠ profiles.user_id — a stale shadow-claim artifact that silently broke loadAuthedProfile() (the function gating every /dashboard/* page) into a permanent dashboard↔onboarding redirect loop. Fixed via the FK's ON UPDATE CASCADE for all 52 rows.

Residual, not fixed (flagged, out of scope for Tranche 2): lesson_status enum is lowercase-only with no rescheduled value (blocks ~7 reporting views); no storage-api container (file uploads won't work locally); migration bookkeeping (schema_migrations) wasn't back-filled for the bulk replay.

c. Manual click-through (5 min)

  • Calendar: sign in as teacher → "Calendar" now appears in the sidebar → /dashboard/calendar shows connect/disconnect status → /dashboard/calendar/conflicts shows an empty state when clean.
  • Locked accounts: as admin, /dashboard?view=admin — a "Locked accounts" card appears only when a profile has locked_until > now(); Unlock clears it and the row disappears.
  • System logs: as admin, /dashboard/logs shows the latest 100 rows with level/prefix filters; teacher/student hitting the URL get redirected to /dashboard.
  • Deleted routes: /dashboard/assignments/templates, /lessons/import, /users/invite, /lessons/<id>/live all render "Page not found".
  • Notifications: /dashboard/notifications — "Mark read" clears a single item, "Mark all read" clears everything; toggling a per-type preference in /dashboard/settings/notifications persists across reload.

3. Edge cases worth knowing about

  • The local Supabase login rate-limiter (15-min lockout after repeated failed/rapid logins) will trip if you re-run the full E2E suite many times back-to-back. Reset via: DELETE FROM auth_rate_limits + UPDATE profiles SET failed_login_attempts=0, locked_until=null for the affected account.
  • Two pre-existing E2E specs (notifications/prefs.spec.ts, seeding a hardcoded student UUID that didn't match the actual fixture account) were fixed to resolve ids dynamically via tests/helpers/seed-ids.ts, consistent with the rest of the suite.
