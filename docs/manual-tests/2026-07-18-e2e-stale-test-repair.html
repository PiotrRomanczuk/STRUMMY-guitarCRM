Date: 2026-07-18  ·  Author: Claude  ·  Branch: fix/e2e-desktop-chrome-failures  ·  Source: docs/analysis/2026-07-18-e2e-desktop-chrome-run.md
Headline: The 50 Playwright failures were not the "AI backend down / DB unseeded" problems the analysis doc claimed. The dominant cause is stale tests running against a redesigned UI (editorial dashboards, a moved AI chat, a rebuilt sign-up form). Most were fixed in test code; a genuine residue needs a seeded DB (which you opted not to provision).

1. What was actually done

Four commits on fix/e2e-desktop-chrome-failures. Quality gates: tsc --noEmit ✅ 0 errors, eslint ✅ 0 errors, every touched spec compiles under playwright test --list. The full browser suite was not executed (needs the prod build + tunnel + optional AI provider — see §2); fixes were verified by matching each spec to the current component source.

The redesign the doc missed

  • /dashboard/ai was replaced — it now shows insight cards + an email generator. The chat moved to /dashboard/ai/chat with a redesigned, test-id-free DOM.
  • The assignment/lesson forms became AssignmentCreateEditorial / LessonFormEditorial — native <select>/<input> with new ids, not the old shadcn combobox test-ids.
  • The dashboard renders editorial views greeting "Good morning…", not "Welcome"; the old TodayLessons/UpcomingLessons cards are no longer mounted.
  • Sign-up: password minimum is now 8 (+letter+number), mismatch copy is "Passwords don't match", duplicate-email copy changed, and a "Go back" button now precedes the form.

2. How to manually verify

a. Static gates (no server needed — reproduces what I ran)

cd strummy
npm run typecheck      # expect: 0 errors
npx eslint components/ai/chat            # expect: clean (chat test-id changes)
npx playwright test --list >/dev/null && echo "all specs compile"

b. Run the repaired specs against the local stack

Same setup as the original run. Categories A/B/C/E and dashboard-states should now pass; the send/response AI assertions run only with a provider. 

# 1. Tunnel to the local stack on uwh
ssh -fN -L 54321:localhost:54321 uwh

# 2. Prod build with the tunnel URL baked in
NEXT_PUBLIC_SUPABASE_LOCAL_URL=http://127.0.0.1:54321 npm run build
NEXT_PUBLIC_SUPABASE_LOCAL_URL=http://127.0.0.1:54321 npx next start -p 3100

# 3. Run the repaired specs (Desktop Chrome)
NEXT_PUBLIC_SUPABASE_LOCAL_URL=http://127.0.0.1:54321 PLAYWRIGHT_BASE_URL=http://localhost:3100 \
  npx playwright test --config=playwright.local-3100.config.ts --project="Desktop Chrome" \
  tests/e2e/ai tests/e2e/auth/sign-up-complete.spec.ts tests/e2e/demo \
  tests/e2e/dashboard/states.spec.ts tests/e2e/cross-role/rls-data-isolation.spec.ts \
  --reporter=line

Expected: green, except assertions that need optional services (below).
