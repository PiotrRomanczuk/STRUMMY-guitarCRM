Bug found and fixed in the RLS test harness itself

lib/testing/rls/seedTwoTeachers.ts (and two other RLS spec files) upserted a profile without user_id. PostgREST's upsert is not a partial merge — every column absent from the payload gets overwritten with its schema default, silently nulling the user_id the handle_new_user trigger had just set, and tripping the ck_shadow_user_id check constraint on the very next query. This was failing every RLS test that uses the two-teacher harness before the fix. Now fixed in all 3 files.

  • songs.rls.test.ts: student can see songs beyond their own lesson-linked set (2 failing assertions) — needs its own investigation, unrelated to ASG-3.
  • profiles.rls.test.ts: teacher can see a deactivated student / another teacher's student by direct id lookup (2 failing assertions) — same, needs dedicated review.
  • core-tables.rls.test.ts: v_teacher_lesson_trends view doesn't exist — this is the already-known lesson_status enum-casing gap from the earlier Tranche 2 migration replay (documented in 2026-07-19-roadmap-tranche2.html).
