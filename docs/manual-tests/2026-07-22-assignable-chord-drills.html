What was built

The chord quiz — previously ship-ready but reachable only by a direct URL behind a "Coming soon" stub — is now something a teacher can direct, and the student's score flows back into the teacher-driven loop.

  • Teacher attaches a chord drill (a set of chords) to any assignment via a chip picker on the create/edit form.
  • Student opens the assignment, taps "Start chord drill", and is deep-linked into the chord quiz seeded with exactly those chords.
  • On finishing, the score (e.g. 8/10) is stamped back onto the assignment via a SECURITY DEFINER RPC (student_complete_chord_drill) — the student's only DB write path — and the assignment is marked completed.
  • Teacher sees the captured score on the assignment detail.
  • The Skills hub (/dashboard/skills) was rebuilt from a stub into a real hub and revealed in the teacher sidebar (CHT-2).

Data model: two nullable jsonb columns on assignments — chord_drill {chord_ids} (teacher-authored) and chord_drill_result {score,total,completed_at} (server-written). Migration supabase/migrations/20260722000001_assignment_chord_drills.sql, already applied to the StudentDevelopment dev stack.

Happy path

  1. Teacher — go to /dashboard/assignments/new. Pick a student, give it a title (e.g. "Open chords check"). Scroll to Chord drill (optional) and click 4–5 chord chips (e.g. C, G, D, Am, Em). Expect: selected chips highlight; a "N chords selected" line appears. Create the assignment.
  2. Teacher — on the assignment detail, the Progress card shows the drill's chords and "Awaiting the student's result."
  3. Student — log in as the student, open the same assignment (Assignments → the new one). The Progress card shows the chords and a "Start chord drill →" button.
  4. Student — click it. Expect: the chord quiz opens titled "Chord Drill" with only the assigned chords, no Random/Review toggle. Answer each question.
  5. Student — on the results screen, see score / total and "Results saved.", plus a "Back to assignment" button. Click it.
  6. Student — the assignment now shows status completed and "Scored N/total · <date>" in the Progress card.
  7. Teacher — reopen the assignment. Expect: the same score is visible, and status is completed.
  8. Teacher — confirm the Skills item now appears in the sidebar → /dashboard/skills is a real hub (not "Coming soon"), linking to the chord quiz (with a "N due" badge if any SRS chords are due).

Edge cases worth checking

Case Expected
Assign a 1-chord drill Quiz still shows 4 answer options (distractors drawn from the full chord library).
Create an assignment with no chords selected No drill; detail shows no chord-drill block. Normal assignment.
Edit an existing drill assignment The previously-selected chords are pre-highlighted; changing them saves.
Open /dashboard/skills/chord-quiz directly (no ?drill=) Normal quiz (random, or Review if SRS chords are due) — unchanged behaviour.
Hit /dashboard/skills/chord-quiz?drill=<someone-else's-id> RLS returns no row → falls through to the normal quiz. No leak. (A tampered drill id at submit is rejected by the RPC's ownership check.)
