Student Detail — At-Risk (health-aware redesign)

Date: 2026-07-23 · Branch: feature/student-detail-at-risk · Route: /dashboard/users/[id]

1. What was built

The existing StudentDetailEditorial was upgraded from a static profile into a health-aware student view, implementing the batch-02 “Student Detail — At Risk / Healthy” mockup. Both the at-risk and healthy states are the same component; the header just renders by status.

  • Health status header — a pill reading On track (<7 days since practice), Needs attention (7–13 days), or At risk (14+ days / never practiced). Reuses the teacher-dashboard at-risk threshold. Below the name: “N days since practice”.
  • Reach-out CTA — a prominent mailto: button that reads Reach out (amber) when the student needs attention, or Message when on track.
  • 14-day practice chart — daily bars with a dashed goal line at 30 min/day; bars that hit the goal read gold. Shows the trailing-7-day total (“this week”).
  • Next lesson card — next upcoming lesson with a Reschedule now link, or a “Schedule lesson” nudge when there is none.
  • Teacher notes card — the most recent lesson’s note (read-only; sourced from lesson notes — there is no dedicated per-student notes store yet).
  • Tabs — Overview (default) / Lessons / Repertoire / Practice Log.

2. How to manually test

  1. Start the app: npm run dev. Sign in as admin (p.romanczuk@gmail.com / test123_admin) or a teacher.
  2. Go to /dashboard/users and open any student → lands on /dashboard/users/<id>.
  3. Header health: a student with no recent practice shows At risk + a Reach out button. A student who practiced in the last few days shows On track + a Message button.
  4. Overview tab: confirm the 14-bar practice chart, the goal line, and the “this week” total; the Next-lesson card (or “No upcoming lesson.” + Schedule link); the Teacher-notes card (or “No teacher notes yet.”).
  5. Reschedule: click Reschedule now → → should open /dashboard/lessons/<lessonId>/edit.
  6. Tabs: click Lessons (full lesson list), Repertoire (songs; staff see a status dropdown per song), Practice Log (raw sessions: date · song · minutes · note).
  7. Reach out: click Reach out / Message → opens a mail draft to the student’s email.

3. Edge cases worth checking

Case Expected
Student with no repertoire / no practice At-risk badge, “No practice logged yet”, empty chart (flat rule ticks), zeroed stats, empty tabs.
Shadow (unclaimed) profile “Unclaimed” badge + Invite / Delete actions still present.
Student who never onboarded Preferences chips line is omitted (no skill/goals row).
Mobile width (< 860px) Overview grid + header stats stack into one column; tabs scroll horizontally.
Dark mode Editorial dark tokens apply (badge/chart colors inherit --success/--warn/--danger).
