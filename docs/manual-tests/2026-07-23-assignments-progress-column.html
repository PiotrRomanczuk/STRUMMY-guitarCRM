Assignments — teacher progress column + student submit polish

Date: 2026-07-23 · Branch: feature/assignments-progress · Scope: editorial assignments list + detail

What was built

Two visible changes, both driven entirely by existing data (the per-assignment checklist JSON already loaded by the list query — no new DB columns, no media upload).

  1. Teacher list — dedicated “Progress” column. The teacher/admin assignments list now shows a fourth column between the student/title and the status pill: a thin completion bar plus a done/total count ( 2/4), derived from the checklist via checklistProgress. Assignments with no checklist show a muted — placeholder (hover title: “No checklist on this assignment”). The column is teacher/admin only — the student list stays three columns (Due / Title / Status); a student sees their own progress in the detail view instead. Previously the compact bar was crammed under the status pill for every role; it now has its own labelled column and header.
  2. Student detail — clearer “hand it in” submit path. The detail Progress card body was extracted into AssignmentSubmitPanel. For an acting student the card is titled “Your practice” and the status action is framed with a “Hand it in” eyebrow plus the line “Tick off what you’ve practised, then mark it complete to send it to your teacher.” A teacher/admin sees the neutral “Progress” / “Update status” framing with no hand-in copy. The underlying checklist tick-offs, chord-drill view, and status transitions are unchanged.

Data / query changes

  • No query change was required: getAssignmentsList already selects checklist and mapRow already computes progress: { done, total } onto every AssignmentRow (from a prior PR). This wave only surfaces that field as a real column.
  • No migration. No new columns. No daily_target / submission_type invented.
