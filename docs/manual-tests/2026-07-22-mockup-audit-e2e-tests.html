⚠ Known blocker — the dev Supabase stack isn't currently seeded. Three of the four implementing agents independently hit this, one confirming it directly against the Supabase Auth REST API (not just browser flakiness): the documented dev test accounts (p.romanczuk@gmail.com, teacher@example.com, student@example.com / student1@example.com) return invalid_credentials on the shared LAN dev stack (192.168.1.75:55321), and profiles has zero rows for those emails. This means none of these new specs — nor any existing spec using loginAs() — can get a real login-gated pass against that stack right now. This is a pre-existing environment condition, not something introduced by this branch. Action needed: run npm run seed against the StudentDevelopment stack on uwh (or otherwise reseed it) before trusting a live E2E run of this branch. The one spec that doesn't require login (landing-page.spec.ts) was verified genuinely green on real infrastructure.
Separate, unrelated finding: running any @mobile/@tablet Playwright project (WebKit engine) against a plain-HTTP local dev server (localhost:3000) fails every sub-resource load with a TLS error — root cause is proxy.ts's middleware unconditionally setting Content-Security-Policy: upgrade-insecure-requests, which forces WebKit to upgrade asset requests to HTTPS even though localhost has no TLS listener. Presumably invisible in CI if CI runs against an HTTPS preview URL. Pre-existing, not caused by this branch — worth a follow-up ticket if local WebKit/mobile E2E runs are expected to work.

How to manually verify

  1. First, reseed the dev stack (see blocker above) — otherwise every login-gated spec below will fail at beforeEach, not because of a real bug.
  2. Checkout the branch and list all changed/new specs: 
    cd /Users/piotr/Desktop/MainCV/strummy
git checkout feature/e2e-test-coverage-mockup-gaps
npx playwright test --list \
  tests/e2e/teacher/backfill-at-risk.spec.ts \
  tests/e2e/mobile/mobile-responsiveness.spec.ts \
  tests/e2e/notifications/inbox.spec.ts \
  tests/e2e/smoke/landing-page.spec.ts \
  tests/e2e/teacher/fretboard.spec.ts \
  tests/e2e/settings/integrations.spec.ts
  3. Run the one spec that's already verified green without needing auth or seeding: 
    npx playwright test tests/e2e/smoke/landing-page.spec.ts --project="Desktop Chrome"
  4. After reseeding, run the rest: 
    npx playwright test \
  tests/e2e/teacher/backfill-at-risk.spec.ts \
  tests/e2e/mobile/mobile-responsiveness.spec.ts \
  tests/e2e/notifications/inbox.spec.ts \
  tests/e2e/teacher/fretboard.spec.ts \
  tests/e2e/settings/integrations.spec.ts \
  --project="Desktop Chrome"
    (Skip WebKit-based mobile/tablet projects until the CSP/localhost TLS issue above is resolved, or point at an HTTPS preview URL instead of localhost.)
  5. Expected: all new/modified cases pass; no existing case regresses.
