Teacher Dashboard — Direction A deltas

Date: 2026-07-23 · Branch: feature/teacher-dashboard-direction-a
Summary

The teacher dashboard already matched Direction A's information architecture (greeting, day spine, needs-attention, week density, roster). This change layers on three additions from the chosen mockup, without redesigning the existing dashboard:

  1. Quick actions card — a 2×2 grid of shortcuts to the four most common create flows: New lesson, Assignment, Add song, Invite student.
  2. Activity feed card ("Recent across your studio") — a new server query aggregating recent student practice sessions, completed assignments, and completed lessons, merged and sorted newest-first, rendered as a compact feed with actor · action · relative time.
  3. Song of the week card — replaces the old song-library summary. Shows the currently active song of the week (title, author, key/capo/tempo/level, teacher note) with an Assign → CTA. Falls back to an empty state when none is selected.

Studio-activity query approach

getStudioActivity(teacherId, now, limit=8) runs a lessons query to find the teacher's distinct student IDs, then three parallel reads bounded to the last 30 days:

  • practice_sessions for those students → "practiced <song>"
  • assignments with status = 'completed' for this teacher → "completed <title>"
  • lessons with status = 'COMPLETED' for this teacher → "finished a lesson"

Rows are mapped to a common StudioActivityItem shape and passed through the pure mergeStudioActivity() helper (flatten → sort by occurredAt desc → slice to limit). The pure merge/sort and the relativeTimeLabel() formatter are unit-tested in isolation; the query itself is tested with a mocked Supabase builder. A per-source query error is logged but does not fail the whole feed.

How to manually test

Setup

cd strummy
npm run dev
# Sign in as the teacher/admin account (see CLAUDE.md dev credentials)
# Open http://localhost:3000/dashboard  (teacher view is the default)

1. Quick actions

  1. Locate the Quick actions card at the top of the right rail.
  2. Confirm four tiles in a 2×2 grid: New lesson, Assignment, Add song, Invite student.
  3. Click each and confirm the destination:
    • New lesson → /dashboard/lessons/new
    • Assignment → /dashboard/assignments/new
    • Add song → /dashboard/songs/new
    • Invite student → /dashboard/users/new
  4. Expected: each route loads its create form; tiles show a gold-border hover.

2. Activity feed

  1. Find the full-width Activity — Recent across your studio card near the bottom.
  2. With seeded data present, confirm rows show: student initials, "Name practiced/completed/finished a lesson …", and a relative time (e.g. 2h, 3d) on the right.
  3. Confirm newest events appear first.
  4. Empty state: a studio with no activity in the last 30 days shows "No studio activity in the last month."

3. Song of the week

  1. Find the Song of the week card (bottom row, next to Roster) with the gold-tint gradient.
  2. If a song of the week is active: confirm the cover initials, title (links to /dashboard/songs/<id>), author, and the KEY / CAPO / ♩tempo / level meta row.
  3. Confirm the Assign → button links to /dashboard/assignments/new.
  4. Empty state: when no SOTW is set, the card shows "No song of the week selected yet." and no Assign button.

4. Dark mode & responsive

  • Toggle dark mode — all new cards use editorial CSS vars, so colors invert cleanly.
  • Narrow the window below ~860px — the hero grid and the 2-column row collapse to a single column.

Known scope decisions / gaps

  • The mockup's "Assigned to N students this week" sub-line on the SOTW card was omitted: assignments have no direct song_id link, so the count is not cheaply derivable. The card shows the teacher note (or a default prompt) instead.
  • The activity feed's filter pills (All / Practice / Lessons / Songs) from the mockup were not implemented — they require a client component; the card is kept server/presentational per the editorial boundary rule. Candidate follow-up.
  • Assign → links to the generic assignment-create page (no song pre-fill, since that page does not yet accept a songId param).
