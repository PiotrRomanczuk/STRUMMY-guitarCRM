Date: 2026-07-23 · Route: /onboarding · integrated to main

1. What was built

The v2/onboarding wizard was extended and restyled to the editorial system, implementing the batch-02 Onboarding mockups.

  • Teacher flow — multi-step: About you → Your studio (name, tagline, timezone, what you teach, default lesson length) → Invite students → Schedule first lesson → Done, with a live-preview rail that reflects entered values.
  • Student flow — current level, goals, daily practice target.
  • New server action app/actions/onboarding-editorial.ts; migration 20260723130000_teacher_settings_and_daily_goal.sql (teacher studio settings + daily goal).

2. How to manually test

  1. npm run dev. Register a fresh account (a brand-new teacher has no role until onboarding) — you land on /onboarding.
  2. Teacher path: pick "I teach guitar", then walk the steps. Type a studio name/tagline/city — confirm the live preview updates. Set default lesson length; reach Invite → Schedule → Done.
  3. Student path: pick "I take lessons"; set level, goals, and daily practice target; finish.
  4. Confirm completion lands you on the correct dashboard and the choices persist (revisit profile / student detail).
