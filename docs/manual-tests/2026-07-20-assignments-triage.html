Date: 2026-07-20 · PR: A of the assignments teacher-POV slice · Branch: feature/assignments-triage

What was built

The assignments list at /dashboard/assignments was a flat, ungrouped, non-interactive list (hard 60-row cap, ordered by creation date). It now supports triage:

  • Status tabs — All / Not started / In progress / Overdue / Completed / Cancelled, each showing a live count.
  • Search by title, sort (Needs attention / Due date / Newest / Recently updated / Title / Status) with an asc/desc toggle, and a per-student filter (teachers/admins only).
  • “Needs attention” default order — overdue first, then soonest due, then newest. All filter/sort state lives in the URL (shareable, back-button friendly).
  • Correct counts. The summary line previously counted a pending status that the app never produces (so it always read 0 and not_started work was omitted). Counts are now derived from the effective status.
  • Effective “overdue”. A past-due assignment still stored as not_started/in_progress (between nightly cron runs) now shows and counts as Overdue everywhere — one source of truth (calculateAssignmentStatus).
  • Cleanup — removed the duplicated “Status” column in the teacher list and the stale pending label/colour entries.
Purely UI + query-layer. No schema change, no change to the status state machine, notifications or crons. RLS still bounds each user to their own rows; filtering/counting happens over that already-scoped set (cap 500).

How to manually test

Setup: npm run dev against local Supabase; sign in.

Dev credentials: teacher p.romanczuk@gmail.com / test123_admin, student emma@strummy.app / Demo2024!.

As a teacher/admin

  1. Go to /dashboard/assignments. Confirm the status tab row with counts.
  2. Confirm default order puts any overdue rows first (red due date). The summary line reads “N total · X not started · … · Z overdue” — no “pending”.
  3. Click each tab. The list filters; the URL gains ?status=…; counts stay stable. The Overdue tab includes any assignment whose due date has passed even if its stored status is still Not started / In progress.
  4. Type in Search and submit → ?q=…; only matching titles show.
  5. Change Sort (e.g. Title) and toggle asc/desc → ?sort=title&dir=…; order updates.
  6. Pick a student in the student filter → ?student=…; only that student’s assignments show. (Verify the dropdown lists only your own students.)
  7. Reload with a filtered URL — state is restored. The “+ New assignment” button still works.
  8. Empty state: filter to something with no matches → “No assignments match these filters.”

As a student

  1. Sign in as the student, open /dashboard/assignments. Confirm the “From your teacher” heading, tabs, sort/search — but no per-student filter and no “New assignment” button.
  2. Confirm you only see your own assignments and can still open a detail page.

Edge cases worth a look

  • An assignment with no due date sorts last under “Needs attention”.
  • A completed/cancelled assignment never flips to Overdue regardless of due date.
  • Deleted (soft deleted_at) assignments never appear (this query now filters them).
