UX Audit Remediation — Manual Test Guide

Date: 2026-07-20 · Branch: fix/ux-audit-remediation (off feature/assignments-triage) · Audit: docs/analysis/2026-07-20-ux-frontend-audit.md

What was done

All findings from the 2026-07-20 frontend UX audit were remediated. In behavioral terms, the app now:

  • Shows hover feedback on clickable rows and chips across dashboard cards, People, Songs, Lessons, Assignments, and song sidebars (new .ed-row/.ed-chip classes in app/editorial-tokens.css — inline styles can't express :hover, so interactive styling moved to CSS). Keyboard focus now draws a gold outline on editorial surfaces; every enabled button shows a pointer cursor again (Tailwind 4 preflight had removed it).
  • Has a working dark mode everywhere. The editorial pages' pre-built dark palette is now activated by the sidebar theme toggle (.dark .theme-editorial selector). Previously the shell went dark while all page content stayed light.
  • Is responsive on phones. Teacher dashboard columns stack below 860px (no more crushed "Needs attention" card with overlapping badges); the Assignments header wraps instead of pushing "+ New assignment" 126px off-screen; songs list rows collapse to a labelled one-line meta instead of bare dashes.
  • Renders chord diagrams for ~40 common chords (was 14) and shows a clean "no chart" placeholder — no more clipped duplicate chord names over "?" boxes.
  • Treats overdue consistently: assignment detail now derives overdue at read time like the list, colors the due date red, and both dashboards surface overdue work (teacher: "Overdue homework" card; student: "Assignments due" card, overdue first).
  • Fixes student-facing gaps: no more "Untitled" ghost songs on the student dashboard (RLS-unreadable rows are filtered), practice time reads "14h 49m" instead of "889m", and the song page shows the student's own progress instead of teacher analytics ("Assigned to 1 students").
  • Improves forms and actions: assignment form validates all fields at once with inline red errors and focuses the first invalid field; list filters apply live (debounced typing, instant selects — the "Filter"/"Apply" buttons are gone); checklist items toggle from the whole row (was a 13px checkbox); "Cancel" is now "Cancel assignment"; only one primary status button.
  • Removes hydration errors on /dashboard/theory and /dashboard/ai/chat (the layout-mode hook now initializes SSR-consistently), gates "Generate AI summary" to completed lessons, restyles Calendar into the editorial shell, hides the ⌘K hint on touch, enlarges the sign-out target, and renames the "With author" status to "Play-along".

Test steps

1. Hover, cursor & focus (desktop, teacher)

Do Expect
Hover rows on Dashboard roster / Songs card, People list, Songs list, Lessons list, Assignments list Row background tints warm beige; cursor is a hand
Hover the level/sort chips on Songs and status tabs on Assignments (inactive ones) Chip border/text darkens
Hover any button (theme toggle, sign out, section headers) Pointer cursor (was arrow)
Tab through a list page with the keyboard Gold focus outline is visible on links/buttons/checkboxes

2. Dark mode

Do Expect
Sidebar footer → moon icon → “Dark”, then visit Dashboard, Songs, a song detail, Assignments, Settings Entire page renders dark (warm near-black paper, cream text, gold accents) — not just the sidebar. Chord diagrams draw in light ink
Switch back to “Light” Everything returns to cream

3. Mobile layout (resize to ~390px or use a phone)

Do Expect
Teacher dashboard at 390px Cards stack in one column; student names and “7d” badges don't overlap; no horizontal scrolling
Assignments at 390px “Templates” and “+ New assignment” fully visible under the title; page does not scroll sideways; red “N overdue assignments need a nudge.” line replaces the old counts caption
Songs at 390px Each row: title + one “Author · Level · Key” line (only values that exist — no floating dashes)
Open the drawer (hamburger) No ⌘K badge in the search field on mobile

4. Chords (song detail — e.g. Wonderwall)

Do Expect
Open Wonderwall as teacher or student Em7, G, Dsus4, A7sus4, Cadd9 all render real diagrams (dots + open-string circles), single name label per card
Open a song with an exotic chord (no shape on file) Dashed box with small “no chart” text — no clipped duplicate name, no “?”

5. Assignments flow

Do Expect
Type in “Search title…” on the list List filters as you type (~350ms), no Enter needed
Open an overdue assignment (e.g. “Clean up timing”, due Jul 18) Pill shows OVERDUE (matches the list), due date is red
Look at the status buttons One solid primary (natural next step), others outlined; cancel says “Cancel assignment” and confirms
Submit the New Assignment form empty Both Student and Title show red inline messages under the fields, first invalid field gets focus, borders turn red; errors clear as you type
As student, tap anywhere on a checklist item row (phone width) Item toggles — the whole row is the tap target; progress bar updates instantly

6. Dashboards surface overdue work

Do Expect
Teacher dashboard “Overdue homework” card lists overdue assignments (student + title + red due date) with “View all →” to the filtered list
Student dashboard “Assignments due” card, overdue first (“Overdue · Jul 11” in red); no “Untitled” songs in the repertoire card; practice time shows hours (“14h 49m”)

7. Role-adapted song page & misc

Do Expect
Open a song as the student Right rail shows “Your practice / Progress” (stage stepper + practice total) — no “Assigned to N students” analytics
Open /dashboard/theory and /dashboard/ai/chat (dev console open) No hydration error
Open a SCHEDULED lesson as teacher No “Generate AI summary” section (appears only on completed lessons)
Open Calendar Editorial header (serif italic “Calendar”, TOOLS eyebrow) — no longer a different-looking app
Settings Email/Role fields visibly muted (disabled look); song statuses read “Play-along” instead of “With author”

Edge cases checked

  • Persisted status='overdue' rows (legacy seed data) are included in both dashboard queries.
  • Unknown chord names never borrow a wrong diagram (unit-tested).
  • Theory stays hidden from nav by design (menuConfig.ts — no seeded content yet); it was flagged as “orphaned” in the audit but is intentional.
