All four Identity blockers are already fixed in open PR #525 (fix/shadow-claim-lifecycle, non-draft, opened 2026-07-14) — which also fixes a twin of IDA-6 in the admin manual-link path that the previous revision missed entirely. The fix is written, tested across all three tiers, and adversarially reviewed. Do not reimplement it.

  1. Unpause the Supabase project — CI is red only because Database Quality fails on project is paused, cascading into Quality Gate. Lint, typecheck, unit, integration, build, CodeQL and Security Audit all pass.
  2. Dismiss the GitGuardian flag — the only hit is STUDENT_PASSWORD = 'e2eClaim123!' in a new Playwright spec. Needs a human, not a code change.
  3. Merge #525 to main before this feature branch. #525's migration is stamped 20260714120000; this branch runs to 20260719000007. Neither is on main yet, so there is no conflict today — but merge them out of order and you inherit a Supabase "migration history out of order" repair job. Re-timestamping #525's migration past 20260719000007 is the alternative.
  4. Rebase this branch onto main afterward — the admin-path twin bug resolves for free.
